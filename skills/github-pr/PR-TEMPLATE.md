# PR 描述模板

## feat 类型（新功能）

```markdown
## 背景

<!-- 一句话说明为什么需要此功能，用户痛点 -->

## 解决方案

<!-- 一句话说明实现思路 -->

## 变更概要

| 模块 | 新增 | 修改 | 删除 | 说明 |
|---|---|---|---|---|
| `crates/xxx` | 2 | 1 | 0 | 新增功能核心逻辑 |

## 代码统计

- 变更文件数：N
- 新增行数：+N
- 删除行数：-N

## 演示效果

<!-- 如有截图，用文字描述关键交互步骤 -->

## 合并注意事项

- **影响功能点**：
- **验证方式**：
- **不兼容变更**：无 / 详细说明
- **回滚方式**：revert 此 PR 即可 / 需额外操作

## 关联

Closes #<issue-number>  <!-- 无关联 Issue 时删除此行 -->
```

---

## fix 类型（问题修复）

```markdown
## 问题描述

<!-- 什么场景下触发，现象是什么 -->

## 根本原因

<!-- 技术层面的原因 -->

## 修复方案

<!-- 改了什么 -->

## 变更文件

| 文件 | 修改说明 | 变更行数 |
|---|---|---|
| `src/xxx.rs` | 修复 xxx | +N -N |

## 合并注意事项

- **影响功能点**：
- **验证方式**：
- **回归风险**：
- **回滚方式**：revert 此 PR 即可 / 需额外操作

## 关联

Closes #<issue-number>  <!-- 无关联 Issue 时删除此行 -->
```

---

## 字段填写规范

| 字段 | 来源优先级 |
|---|---|
| 背景 / 问题描述 | prd.md > commit-info.md > git log |
| 解决方案 / 根本原因 | trd.md > commit-info.md > git log |
| 变更概要 / 变更文件 | commit-info.md > `git diff --stat` |
| 代码统计 | commit-info.md > `git diff --shortstat` |
| 演示效果 | 用户提供的截图说明文字 |
| 合并注意事项 | commit-info.md > trd.md > 人工分析 |
